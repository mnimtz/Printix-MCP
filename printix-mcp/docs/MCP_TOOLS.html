Printix MCP Server — Tool Reference

Complete catalogue of MCP tools exposed by the Printix MCP Server for AI assistants.

Version: 6.7.107  |  Transport: Streamable HTTP (/mcp for claude.ai) + SSE (/sse for ChatGPT)  |  Auth: OAuth 2.0 Bearer Token (per tenant)  |  Public Endpoint: https://mcp.printix.cloud

1. Status & Info

ToolDescription
printix_status()Returns server version, configured tenant, OAuth app health. The sanity-check tool — always call this first when debugging.

2. Printers & Queues

ToolDescription
printix_list_printers(search, page, size)List printers in the tenant. Supports free-text search (name, model, location).
printix_get_printer(printer_id, queue_id)Full detail of a single printer/queue pair, including capabilities.

3. Print Jobs

ToolDescription
printix_list_jobs(queue_id, page, size)Current print jobs in the tenant. Optional filter by queue.
printix_get_job(job_id)Detail of one print job (status, owner, size, creation time).
printix_submit_job(...)Submit a new print job. Returns upload URL for the 3-step flow (submit → upload → complete).
printix_complete_upload(job_id)Finalize a print job after the file has been uploaded to the pre-signed URL.
printix_delete_job(job_id)Cancel or delete a print job.
printix_change_job_owner(job_id, new_owner_email)Transfer Secure-Print ownership to another user (e.g. for delegation).

4. Card Management (Printix Cloud)

ToolDescription
printix_list_cards(user_id)All cards of one user, enriched with local DB mappings (raw UID, transform profile, notes, decoded hex/decimal variants).
printix_search_card(card_id, card_number)Lookup a single card by Printix ID or physical card number.
printix_register_card(user_id, card_number)Register a card directly in Printix (skips the local mapping layer).
printix_delete_card(card_id)Remove a card from Printix.

5. Card Management (Local DB)

ToolDescription
printix_list_card_profiles()All transform profiles — built-in vendor presets (HP, Konica, Ricoh etc.) plus tenant-custom profiles.
printix_get_card_profile(profile_id)Single profile with its rules JSON (how a raw UID gets transformed into the Printix secret).
printix_search_card_mappings(search, printix_user_id)Full-text search across locally stored mappings.
printix_get_card_details(card_id, card_number)Enriched single-card view combining Printix API + local DB data.
printix_decode_card_value(card_value)Decode a raw Printix base64 secret into hex / decimal / reversed variants — useful for debugging.
printix_transform_card_value(card_value, ...)Dry-run: apply a transform profile to a raw UID without registering.
printix_get_user_card_context(user_id)One-shot combo: user details + all cards (enriched) + all local mappings for that user.

6. Users

ToolDescription
printix_list_users(role, query, page, page_size)List users. Default role='USER'. Call again with 'GUEST_USER' for guest accounts.
printix_get_user(user_id)Full user detail including attributes and group memberships.
printix_create_user(email, display_name, pin, password)Create a guest user account. PIN must be exactly 4 digits.
printix_delete_user(user_id)Delete a guest user. Managers can't be deleted via API.
printix_generate_id_code(user_id)Issue a fresh 6-digit ID code for card-reader fallback authentication.

7. Groups

ToolDescription
printix_list_groups(search, page, size)List user groups (used for printer assignment).
printix_get_group(group_id)Single group with members and permissions.
printix_create_group(name, external_id)Create a group. external_id typically matches your directory source.
printix_delete_group(group_id)Delete a group.

8. Workstations

ToolDescription
printix_list_workstations(search, page, page_size)List endpoints with Printix Client installed. Filter by hostname or user.
printix_get_workstation(workstation_id)Workstation detail — OS, last seen, online state, associated user.

9. Sites & Networks

ToolDescription
printix_list_sites(search, page, size)List sites (logical locations like "HQ Copenhagen").
printix_get_site(site_id)Site detail with networks and printers.
printix_create_site(name, path, ...)Create a new site node in the organizational tree.
printix_update_site(site_id, ...)Modify site metadata.
printix_delete_site(site_id)Delete a site (requires empty networks).
printix_list_networks(site_id, page, size)List networks (subnets where printers live). Optionally scoped to a site.
printix_get_network(network_id)Network detail including CIDR, SNMP config, printers.
printix_create_network(name, ...)Create a network (subnet + SNMP binding).
printix_update_network(network_id, ...)Modify network.
printix_delete_network(network_id)Delete a network.

10. SNMP Configurations

ToolDescription
printix_list_snmp_configs(page, size)All SNMP configurations (v2c/v3 credentials for printer discovery).
printix_get_snmp_config(config_id)Single SNMP config.
printix_create_snmp_config(name, ...)Create an SNMP config (v2c with community or v3 with user/auth/priv).
printix_delete_snmp_config(config_id)Delete SNMP config.

11. Reporting — Queries

ToolDescription
printix_reporting_status()Health of the SQL Server data source, last ingest time, row counts.
printix_query_print_stats(...)Raw print-volume aggregation. Group by user / printer / site / day.
printix_query_cost_report(...)Cost calculation with configurable €/page for mono, color, sheet.
printix_query_top_users(start_date, end_date, top_n, metric, ...)Ranking of most active users. Metrics: pages, cost, jobs, color_pages.
printix_query_top_printers(start_date, end_date, top_n, metric, ...)Ranking of most-used printers — same metrics as top_users.
printix_query_anomalies(...)Detect usage spikes or drops vs. historical baseline.
printix_query_trend(...)Time-series for print volume, grouped by day / week / month.
printix_query_any(sql_like_spec)Generic escape hatch — execute a saved-template spec directly.

12. Reporting — Templates & Schedules

ToolDescription

printix_list_report_templates()All saved report templates in the tenant.
printix_get_report_template(report_id)Template detail with its query spec and design options.
printix_delete_report_template(report_id)Delete template.

printix_send_test_email(recipient)Verify SMTP configuration / email delivery.
printix_schedule_report(...)Attach a cron-like schedule to a template (daily/weekly/monthly).

printix_delete_schedule(report_id)Remove schedule from a template.
printix_update_schedule(...)Modify an existing schedule.

13. Reporting — Design & Preview

14. Demo Data

Conventions

  • Return format: every tool returns a JSON-encoded string.
  • Error shape: Printix API errors return {"error": true, "status_code": N, "message": "...", "error_id": "..."}. Other errors return {"error": "message"}.
  • Tenant isolation: each request resolves its tenant via Bearer token. Tools operate only on that tenant's data.
  • Pagination: listing tools accept page (0-based) and page_size / size (default 50, max 200).
  • Date format: YYYY-MM-DD for all date arguments.
  • Cost parameters: cost_per_mono, cost_per_color, cost_per_sheet are per page/sheet in the tenant's currency; defaults reflect rough EU averages.

Integration Patterns

"What cards does Marcus have?"

printix_find_user("marcus")
    → user_id
printix_get_user_card_context(user_id)
    → cards + enriched local data

"Who printed the most last month?"

printix_top_users(days=30, limit=10, metric="pages")

"User X can't print — help me debug"

printix_diagnose_user("x@company.com")
    → findings list with severities + suggestions

"Onboard a new employee"

printix_onboard_user(email=..., display_name=..., groups="grp1,grp2")
printix_generate_id_code(user_id)   # optional
# card registration happens in iOS/Web client

"How did print volume change vs last month?"

printix_compare_periods(days_a=30, days_b=30)

"Just print this file for Marcus"

printix_quick_print(
    recipient_email="marcus@company.com",
    file_url="https://.../document.pdf",
    filename="document.pdf")
